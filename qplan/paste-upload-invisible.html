3. useEffect reset state — root cause của "file biến mất"

Khi !tempFile, có useEffect(res) đồng bộ component state với V2 file list:

useEffect(() => {
  if (files !== undefined && !tempFile) {
    // Tìm file_path trong V2 list
    const match = files.find(f => f.path === file_path);
    if (!match) {
      handleOnNewValue({ value: "", file_path: "" }) // ← RESET về rỗng!
    }
  }
}, [files, value, file_path]);

Timeline với V1 (cũ):

  1. Upload file → V1 API → không tạo UserFile → không có trong V2 cache
  2. handleOnNewValue({ value: "file.png", file_path: "flowId/ts_file.png" })
  3. useEffect fires → V2 cache không có file → reset state về rỗng
  4. File biến mất khỏi component! ❌

Timeline với V2 (fix):

  1. Upload file → V2 API → tạo UserFile → optimistic update V2 cache
  2. handleOnNewValue({ value: data.name, file_path: "userId/file.png" })
  3. useEffect fires → V2 cache có file với path "userId/file.png" → match thành công
  4. State preserved! ✅

Sửa 3 — Guard paste handler tránh duplicate khi modal mở

Thêm guard trong handleFileChange:

const handleFileChange = useCallback(
  (event: React.ChangeEvent<HTMLInputElement> | ClipboardEvent) => {
    if ("clipboardData" in event) {
      // Nếu modal file manager đang mở, để DragFilesComponent xử lý
      const dialog = (event.target as HTMLElement)?.closest('[role="dialog"]');
      if (dialog) return;
      
      const items = event.clipboardData?.items;
      if (!items) return;
      for (let i = 0; i < items.length; i++) {
        const file = items[i].getAsFile();
        if (file) {
          uploadFile(file);
          return;
        }
      }
    }
  },
  [uploadFile],
);
