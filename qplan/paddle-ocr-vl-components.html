1. Tổng quan Scope

Tạo 2 component mới dạng Langflow bundle trong thư mục src/bundles/, mỗi component nằm trong bundle riêng:

  • OCR Paddle (src/bundles/paddleocr/) — OCR đa ảnh dùng PaddleOCR + PP-DocLayoutV3, xuất ra Markdown
  • VL Images (src/bundles/vlimages/) — Mô tả ảnh dùng Qwen3.5-2B chạy trên vLLM, xuất ra Markdown

Cả hai component đều hoạt động offline, chạy model trên máy local, không gọi API cloud.

2. SRS — OCR Paddle Component

2.1 Mục đích

Cho phép người dùng upload hoặc paste nhiều ảnh, component sẽ dùng PaddleOCR để nhận dạng chữ (text detection + recognition) kết hợp PP-DocLayoutV3 để phân tích layout (tiêu đề, đoạn văn, bảng biểu, hình ảnh), sau đó tổng hợp thành văn bản Markdown có cấu trúc.

2.2 Công nghệ nền tảng

  • PaddleOCR (paddleocr package) — text detection + recognition engine
  • PP-DocLayoutV3 — document layout analysis model (safetensors từ HuggingFace)
  • Output format: Markdown (có cấu trúc heading, paragraph, list, code block, table)

2.3 Functional Requirements

2.4 Non-functional Requirements

  • NFR-01 (Isolation): PaddleOCR (PyTorch) chạy trong subprocess riêng, giống pattern Docling _CHILD_SCRIPT trong file.py — tránh OOM trên main process
  • NFR-02 (Memory): Giải phóng model sau mỗi lần xử lý để tránh rò rỉ GPU memory
  • NFR-03 (Batch): Xử lý nhiều ảnh tuần tự để kiểm soát memory
  • NFR-04 (Dependency): paddleocr >= 2.8 trong dependencies của bundle

2.5 Input / Output Schema

  - paste_images: StrInput (base64 encoded image strings, advanced, tool_mode)
  - lang: DropdownInput (ngôn ngữ OCR, options=["Auto","chinese","english","vietnamese","japanese","korean","french","german"], value="Auto")
  - timeout: IntInput (timeout per image, default=300, advanced)

3. SRS — VL Images Component

3.1 Mục đích

Cho phép người dùng upload nhiều ảnh, component dùng Qwen3.5-2B (VL model) chạy trên vLLM để mô tả nội dung từng ảnh và trả về kết quả dạng Markdown.

3.2 Công nghệ nền tảng

  • Model: unsloth/Qwen3.5-2B-MTP-GGUF (file: Qwen3.5-2B-UD-IQ2_M.gguf)
  • Engine: vLLM (OpenAI-compatible API) — tận dụng VllmComponent có sẵn
  • Quantization: Q4_0 (4-bit)
  • Context window: 50K tokens
  • Max output: 2000 tokens
  • Mode: no-think (disable reasoning/thinking tokens)
  • Output format: Markdown

3.3 Functional Requirements

FR-VL-02Paste ảnh từ clipboard dạng base64 stringP1
FR-VL-03Kết nối đến vLLM server (OpenAI-compatible) — user tự start vLLM server riêngP0

FR-VL-07Mỗi ảnh → 1 mô tả Markdown (heading với tên file, nội dung description)P0

3.4 Non-functional Requirements

  • NFR-VL-01: Không giữ model trong memory — dùng OpenAI-compatible API qua HTTP
  • NFR-VL-02: Kế thừa pattern từ VllmComponent (langchain_openai ChatOpenAI)
  • NFR-VL-03: Timeout configurable cho mỗi request
  • NFR-VL-04: Hỗ trợ system prompt tùy chỉnh để control output format

3.5 vLLM Server Setup Instructions (user cần chạy trước)

# User self-starts vLLM server trước khi dùng component
vllm serve unsloth/Qwen3.5-2B-MTP-GGUF \
  --quantization gguf \
  --max-model-len 50000 \
  --max-num-seqs 4 \
  --port 8000 \
  --trust-remote-code

# Component sẽ connect tới: http://localhost:8000/v1

4.2 Bundle pattern (tham khảo bundle arxiv/docling)

Mỗi bundle có:

  • pyproject.toml — dependencies (paddleocr / openai+langchain) + entry point
  • lfx.toml — khai báo plugin cho CLI phát hiện
  • components/__init__.py — export component classes
  • components/<name>/__init__.py — import component modules

5. Scope Boundary

✅ In Scope

  • IN Tạo 2 bundle mới: paddleocr và vlimages
  • IN Component OCR Paddle: xử lý ảnh → PaddleOCR + PP-DocLayoutV3 → Markdown
  • IN Component VL Images: ảnh → vLLM (Qwen3.5-2B) → Markdown description
  • IN FileInput multiple ảnh (upload) + StrInput base64 (paste)
  • IN Subprocess isolation cho PaddleOCR (tránh OOM)
  • IN No-think mode cho VL model (disable reasoning tokens)
  • IN User tự start vLLM server riêng (component chỉ gọi API)

❌ Out of Scope

  • OUT Không modify bất kỳ file nào ngoài src/bundles/paddleocr/ và src/bundles/vlimages/
  • OUT Không sửa frontend icons, lazyIconImports, hay bất kỳ file React nào
  • OUT Không sửa src/lfx/ hoặc core engine
  • OUT Không tự động start/stop vLLM server
  • OUT Không tải model weights tự động (user tự quản lý)
  • OUT Không hỗ trợ GPU/CPU device selection (dùng mặc định của PaddlePaddle/vLLM)
