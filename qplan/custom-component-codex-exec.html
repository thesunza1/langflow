1. Software Requirements Specification

1.1 Purpose

Custom component Codex Exec cho phép nhập nhiều đoạn text + đường dẫn thư mục, ghép text lại, và chạy lệnh codex exec --dangerously-bypass-approvals-and-sandbox "<text>" --DIR <dir>. Kết quả trả về làm output.

1.2 Functional Requirements

  1. FR-1: MultilineInput — mỗi dòng là 1 prompt, user có thể thêm/xoá dòng tự do mà không giới hạn số lượng.
  2. FR-2: 1 directory input — đường dẫn thư mục làm việc.
  3. FR-3: 1 output — trả về kết quả từ lệnh dạng Message/Data.
  4. FR-4: Ghép các text inputs bằng dấu cách: "input1 input2 input3 ...".
  5. FR-5: Chạy lệnh: codex exec [--dangerously-bypass-approvals-and-sandbox] "<text>" --DIR <dir> — flag bypass sandbox là optional (boolean input, mặc định OFF).
  6. FR-6: Capture stdout + stderr, trả về kết quả gộp.
  7. FR-7: Bắt lỗi: directory không tồn tại, lệnh thất bại, timeout.

1.3 Non-functional Requirements

  • Dùng subprocess.run với shell=False, capture_output=True, text=True (theo pattern có sẵn trong codebase).
  • Check codex có trên PATH trước khi chạy.
  • Validate directory path tồn tại.
  • Timeout 6000s mặc định, configurable.
  • Output trả về Data thay vì Message để đồng bộ với các component mẫu (PythonREPLComponent).
  • Flag --dangerously-bypass-approvals-and-sandbox được điều khiển bằng boolean input, mặc định OFF.
  • Output bị truncate ở 100k ký tự để tránh UI overload.
  • Set self.status ở mọi nhánh (thành công, lỗi, timeout) để UI hiển thị trạng thái.
  • Dùng self.log() để ghi log khi có lỗi.
  • Tuân thủ đúng pattern component: class → inputs → outputs → method.

    from .codex_exec import CodexExecComponent

_dynamic_imports = {
    "CodexExecComponent": "codex_exec",
}

__all__ = ["CodexExecComponent"]

def __getattr__(attr_name: str) -> Any:
    if attr_name not in _dynamic_imports:
        msg = f"module '{__name__}' has no attribute '{attr_name}'"
        raise AttributeError(msg)
    result = import_mod(attr_name, _dynamic_imports[attr_name], __spec__.parent)
    globals()[attr_name] = result
    return result

def __dir__() -> list[str]:
    return list(__all__)

5. Component Design

from lfx.custom.custom_component.component import Component
from lfx.io import MessageTextInput, StrInput, Output
from lfx.schema.message import Message
import subprocess
import shutil
from pathlib import Path

from __future__ import annotations

from lfx.custom.custom_component.component import Component
from lfx.io import BoolInput, MessageTextInput, MultilineInput, Output
from lfx.schema.data import Data
import subprocess
import shutil
from pathlib import Path

class CodexExecComponent(Component):
    display_name = "Codex Exec"
    description = "Combine multiple text inputs and execute via Codex CLI."
    documentation = ""
    icon = "terminal"
    name = "CodexExec"

    inputs = [
        # MultilineInput: mỗi dòng là 1 prompt, user có thể thêm/xoá dòng tự do
        # input_types=["Message"] cho phép kéo kết nối từ node khác vào
        MultilineInput(
            name="prompts",
            display_name="Prompts",
            info="Each line is one prompt. Chúng sẽ được ghép lại với nhau bằng dấu cách. Có thể kéo từ node khác vào.",
            value="Prompt 1\nPrompt 2",
            required=True,
        ),
        MessageTextInput(
            name="directory",
            display_name="Directory",
            info="Working directory for Codex CLI. Path must exist.",
            required=True,
        ),
        BoolInput(
            name="bypass_sandbox",
            display_name="⚠️ Bypass Sandbox",
            info="⚠️ DANGER: Bypasses Codex approval & sandbox. Only enable if you trust the prompts. Disabled by default.",
            value=False,
            advanced=True,
        ),
        MessageTextInput(
            name="timeout",
            display_name="Timeout (s)",
            value="6000",
            advanced=True,
        ),
    ]

    outputs = [
        Output(display_name="Result", name="result", method="execute"),
    ]

    def execute(self) -> Data:
        self.status = "Running..."

        # 1. Ghép text — mỗi dòng trong MultilineInput là 1 prompt
        prompts = [p.strip() for p in self.prompts.split("\n") if p.strip()]
        if not prompts:
            self.status = "Error: no prompts"
            return Data(data={"error": "At least one prompt is required."})

        combined = " ".join(prompts)

        # 2. Validate directory — resolve path để tránh path traversal
        try:
            dir_path = Path(self.directory).resolve()
            if not dir_path.is_dir():
                self.status = "Error: directory not found"
                return Data(data={"error": f"Directory '{self.directory}' not found."})
        except Exception as e:
            self.status = "Error: invalid path"
            return Data(data={"error": f"Invalid directory path: {e!s}"})

        # 3. Check codex CLI
        codex_path = shutil.which("codex")
        if not codex_path:
            self.status = "Error: codex not found"
            return Data(data={"error": "'codex' command not found on PATH."})

        # 4. Build command
        cmd = ["codex", "exec"]
        if self.bypass_sandbox:
            cmd.append("--dangerously-bypass-approvals-and-sandbox")
        cmd += [combined, "--DIR", str(dir_path)]

        # 5. Run
        try:
            self.log(f"Running: {' '.join(cmd[:3])}...")
            result = subprocess.run(
                cmd,
                capture_output=True,
                text=True,
                timeout=int(self.timeout),
                shell=False,
            )
            # Tách stdout/stderr riêng
            output = result.stdout.strip()
            if result.stderr.strip():
                output += "\n\n[STDERR]\n" + result.stderr.strip()
            # Truncate nếu quá dài
            max_len = 100000
            if len(output) > max_len:
                output = output[:max_len] + "\n\n... [Output truncated]"
            self.status = f"Done (exit code {result.returncode})"
            return Data(data={"result": output})
        except subprocess.TimeoutExpired:
            self.status = "Error: timeout"
            return Data(data={"error": f"Command timed out after {self.timeout}s."})
        except Exception as e:
            self.log(f"Execution error: {e!s}")
            self.status = "Error"
            return Data(data={"error": f"Execution error: {e!s}"})

8. Multi-Perspective Review

Review từ 3 góc nhìn: Backend/Architecture, Security/DevOps, UX/Product.

8.1 🔧 Backend & Architecture Review

#FindingSeverityAction
1 Thiếu `from __future__ import annotations` — tất cả component đều có dòng này ở đầu file. MED Thêm vào đầu codex_exec.py.
2 Output type là `Message` — các component mẫu (PythonREPLComponent) dùng `Data` chứ không dùng `Message` trực tiếp. Dùng `Message` vẫn được nhưng cần kiểm tra handle output. MED Đổi output method trả về `Data` cho đồng bộ, hoặc giữ `Message` nếu test thấy handle hoạt động.
3 Module init thiếu `from __future__ import annotations`. LOW Thêm vào đầu __init__.py.
4 Không xử lý prompts rỗng — nếu connected node trả về empty string. LOW Filter empty prompts sau khi strip().
5 Thiếu `documentation` field — các component khác đều có. LOW Thêm documentation = "".
6 Import `StrInput` không dùng. OK Bỏ khỏi imports.
7 Category của component chưa xác định — cần biết nó nằm ở group nào trong sidebar. MED Kiểm tra xem category được xác định bởi folder name hay field riêng.

8.2 🛡️ Security & DevOps Review

#FindingSeverityAction
1 Flag `--dangerously-bypass-approvals-and-sandbox` hardcoded — đây là flag nguy hiểm, bypass toàn bộ sandbox của Codex. HIGH Thêm boolean input để user chủ động bật/tắt, kèm cảnh báo trong info.
2 Directory path injection — user input trực tiếp thành --DIR <path>. MED Dùng Path(self.directory).resolve(), validate path tồn tại, kiểm tra absolute.
3 Shell injection — dù dùng shell=False an toàn, text input vẫn có ký tự đặc biệt. MED shell=False với list args là đủ an toàn. Ghi chú cho user.
4 stderr gộp với stdout — user khó phân biệt error vs output. LOW Tách riêng, append error message nếu có lỗi.
5 Không log output khi lỗi — khó debug. LOW Thêm self.log() ghi stderr khi fail.
6 Không check `codex` version. LOW Ghi minimum version requirement trong doc.

8.3 🎨 UX & Product Review

#FindingSeverityAction
1 Không có dynamic add input — FR-1 yêu cầu "2 hoặc nhiều input" nhưng design chỉ fixed 4 input. MED Dùng MultilineInput (mỗi line = 1 prompt) hoặc TableInput để user thêm dòng linh hoạt.
2 Thiếu placeholder text — các input nên có info để hướng dẫn user. MED Thêm info="..." vào mỗi MessageTextInput.
3 Thiếu progress indicator — user không biết component đang chạy hay treo. MED Set self.status = "Running..." trước exec, "Done" sau.
4 Output raw text có thể rất dài — gây overload UI. LOW Truncate output ở 100k ký tự.
5 Không có cách test component trước khi dùng thật. LOW Thêm test case với input mẫu trong Test Plan.
