Codex Exec — Cải thiện Output

Xử lý output bị lặp (think + answer) và tăng/mở limit output

1. Phân tích hiện trạng

1.1 Vấn đề output bị lặp

codex exec output ra stdout gồm nhiều phần:

  • Header — thông tin phiên bản, model, session id
  • hook events — SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, Stop
  • Tool execution logs — các lệnh shell đã chạy và kết quả
  • Thinking/reasoning — quá trình suy nghĩ của agent (dòng bắt đầu bằng codex)
  • Final answer — câu trả lời cuối cùng

Hiện tại component lấy toàn bộ stdout → chứa cả thinking traces, không phải là output hữu ích cho flow downstream.

Giải pháp: Dùng flag --output-last-message <file> có sẵn của Codex CLI — flag này chỉ ghi câu trả lời cuối cùng của agent vào file, bỏ qua toàn bộ thinking/tool logs.

1
codex exec "prompt" --output-last-message /tmp/msg.txt
2
Codex CLI chạy, stdout vẫn ra đầy đủ (nhưng ta ignore)
3
File /tmp/msg.txt chỉ chứa final answer
4
Đọc file → trả về làm output

1.2 Vấn đề output bị giới hạn

Code hiện tại:

max_len = 100000
if len(output) > max_len:
    output = output[:max_len] + "\n\n... [Output truncated]"

Giới hạn 100.000 ký tự. Sau khi chuyển sang dùng --output-last-message, output sẽ nhỏ hơn (chỉ final answer, không còn thinking traces), nhưng user vẫn muốn tăng/mở limit.

Cách xử lý:

  • Thêm input max_output_length dạng số, giá trị mặc định 500.000, advanced=true
  • Nếu user set = 0 hoặc âm → bỏ qua truncation (unlimited)

2. Yêu cầu chức năng

FR-01: Output chỉ chứa final answer

  • Không còn thinking traces, tool logs, hook events trong output của component
  • Output là message text chứa nội dung câu trả lời cuối cùng từ Codex CLI

FR-02: Cấu hình độ dài output

  • Thêm input max_output_length (advanced) kiểu số, mặc định 500000
  • Giá trị ≤ 0 = unlimited (bỏ qua truncation)

5. Implementation details

Input mới:

IntInput(
    name="max_output_length",
    display_name="Max Output Length",
    info="Maximum output length in characters. Set to 0 for unlimited.",
    value=500000,
    advanced=True,
)

Logic execute() mới:

import tempfile

# Thay vì capture_output=True, dùng --output-last-message
with tempfile.NamedTemporaryFile(delete=False, suffix=".md") as tmp:
    output_path = tmp.name

cmd = ["codex", "exec"]
if self.bypass_sandbox:
    cmd.append("--dangerously-bypass-approvals-and-sandbox")
cmd += [combined, "-C", str(dir_path), "--output-last-message", output_path]

result = subprocess.run(cmd, ..., stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL)

# Đọc final answer từ file
output = Path(output_path).read_text(encoding="utf-8") if Path(output_path).exists() else ""

# Xoá file temp
Path(output_path).unlink(missing_ok=True)

# Truncate nếu cần
if output and hasattr(self, "max_output_length"):
    max_len = int(self.max_output_length)
    if max_len > 0 and len(output) > max_len:
        output = output[:max_len] + "\n\n... [Output truncated]"
