qplan: Ẩn Component Categories trong Sidebar

Giảm số lượng component hiển thị → giảm render → giảm lag khi kéo thả.

In scope: Chỉ sửa frontend code để filter 5 category khỏi danh sách component sidebar. Không sửa backend, không sửa logic flow, không ảnh hưởng đến component đã có trên canvas.
Out of scope: Xoá component khỏi hệ thống, sửa backend API, thay đổi behavior của flow hiện có, tối ưu các phần khác ngoài sidebar.

1. Phân tích

Langflow sidebar hiển thị component categories từ API response. Các categories được render bởi:

  • SidebarItemsList trong flowSidebarComponent/index.tsx
  • Mỗi category là một group item có thể expand/collapse
  • Component trong mỗi category được render dưới dạng draggable items
  • Càng nhiều component → DOM càng lớn → re-render càng nặng khi kéo thả

Các categories muốn ẩn:

  • Models & Agents — nhiều sub-components, chiếm nhiều DOM
  • LLM Operations — tương tự
  • Utilities — ít dùng khi kéo flow cơ bản
  • Prototypes — experimental components
  • Tools — function-calling tools

2. Functional Requirements

  1. FR-1: Cho phép bật/tắt hiển thị từng category riêng lẻ trong sidebar
  2. FR-2: Trạng thái bật/tắt được lưu vào localStorage, giữa các lần reload
  3. FR-3: Categories bị tắt sẽ ẩn hoàn toàn khỏi danh sách (không chỉ collapse)
  4. FR-4: Không ảnh hưởng đến component đã có trên canvas
  5. FR-5: UI toggle nằm gọn trong sidebar, dễ thao tác

3. Non-functional Requirements

  • Performance: Giảm số lượng DOM nodes khi sidebar mở — mục tiêu giảm 40-60% component items
  • UX: Không cần reload page khi bật/tắt
  • Maintainability: Dùng cơ chế tương tự showLegacy/showBeta hiện tại

5. Implementation Plan

5.1 File cần sửa

  • src/frontend/src/pages/FlowPage/components/flowSidebarComponent/components/featureTogglesComponent.tsx
    — Thêm toggles cho từng category
  • src/frontend/src/pages/FlowPage/components/flowSidebarComponent/index.tsx
    — State + filter logic cho hidden categories
  • (Có thể tạo helper mới) src/frontend/src/pages/FlowPage/components/flowSidebarComponent/helpers/apply-category-filter.ts
    — Helper function để lọc categories

5.2 Flow xử lý

  1. User toggle category OFF trong sidebar
  2. Lưu trạng thái vào localStorage (key: hiddenCategories)
  3. Sidebar re-render, filter bỏ category khỏi danh sách
  4. DOM giảm, re-render nhẹ hơn khi kéo thả

5.3 Code pattern

// featureTogglesComponent.tsx — thêm toggle:
{
  label: "Models & Agents",
  category: "Models & Agents",
  checked: !hiddenCategories.includes("Models & Agents"),
  onChange: (v) => toggleCategory("Models & Agents"),
  testId: "sidebar-hide-models-agents",
}

// applyHiddenCategoryFilter.ts:
export const applyHiddenCategoryFilter = (filteredData, hiddenCategories) => {
  return Object.fromEntries(
    Object.entries(filteredData)
      .filter(([category]) => !hiddenCategories.includes(category))
  );
};

// index.tsx — trong useMemo dataFilter:
if (hiddenCategories.length > 0) {
  filteredData = applyHiddenCategoryFilter(filteredData, hiddenCategories);
}

6. Testing

  • Toggle từng category → verify category biến mất khỏi sidebar
  • Reload page → verify trạng thái được restore từ localStorage
  • Flow có component từ category đã ẩn → verify flow vẫn chạy bình thường
  • Toggle all categories off → verify sidebar chỉ còn search + toggles
