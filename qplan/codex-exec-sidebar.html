Codex Exec — Hiển thị trong Components Sidebar

Kế hoạch bổ sung component "Codex Exec" vào danh sách components trong sidebar

1. Phân tích hiện trạng

Component đã tồn tại trong codebase:

  • 📁 src/lfx/src/lfx/components/codex_exec/codex_exec.py — định nghĩa class CodexExecComponent
  • 📁 src/lfx/src/lfx/components/codex_exec/__init__.py — dynamic import mapping
  • 📁 src/lfx/src/lfx/components/__init__.py — registered là "codex_exec": "__module__"
  • 📁 src/lfx/src/lfx/_assets/component_index.json — có trong prebuilt index với đầy đủ template
  • 📁 src/lfx/src/lfx/tests/unit/components/test_codex_exec_component.py — có unit test

Nguyên nhân gốc — không hiển thị trong sidebar:

Frontend dùng danh sách SIDEBAR_CATEGORIES tại src/frontend/src/utils/styleUtils.ts (dòng 314) để lọc whitelist các category được hiển thị. codex_exec không có trong danh sách này, nên bị lọc bởi:

CATEGORIES.some((cat) => cat.name === categoryName)

trong categoryGroup.tsx dòng 48.

2. Yêu cầu chức năng

FR-01: Hiển thị category "Codex Exec" trong sidebar

  • Mô tả: Component "Codex Exec" phải xuất hiện trong danh sách components của sidebar.
  • Vị trí: Sidebar → Components → (trong list các category).
  • Icon: Terminal (lucide icon).
  • Thứ tự: Xuất hiện ở cuối danh sách category.

FR-02: Tương tác đầy đủ

  • Kéo-thả (drag & drop) được từ sidebar vào flow canvas.
  • Hiển thị đúng tên, mô tả, icon khi ở trong canvas.
  • Các input fields (Prompt 1-4, Directory, Bypass Sandbox, Timeout) hiển thị đúng trong node inspector.
  • Output "Result" xuất hiện ở output handle.

FR-03: Search hoạt động

  • Gõ "codex" hoặc "exec" trong thanh tìm kiếm sidebar phải tìm thấy component.

3. Yêu cầu phi chức năng

  • Hiệu suất: Không ảnh hưởng đến thời gian khởi động — component đã được index sẵn.
  • Bảo trì: Chỉ cần thêm 1 entry vào mảng SIDEBAR_CATEGORIES. Không thay đổi backend.
  • i18n: Sử dụng translation key, không hardcode tiếng Anh.
  • UI consistency: Theo đúng pattern của các category khác (display_name, name, icon).

7. Scope boundary

✅ Trong scope

  • Thêm codex_exec vào SIDEBAR_CATEGORIES
  • Thêm i18n translation key
  • Kiểm tra drag & drop hoạt động
  • Kiểm tra search tìm thấy component

❌ Ngoài scope

  • Sửa logic backend component
  • Sửa code của CodexExecComponent
  • Thêm feature mới cho component
  • Tái cấu trúc sidebar
  • Xử lý bundle/extension mechanism
  • Sửa các component khác

8. Risks

  • Low Translation key mới có thể bị thiếu nếu không update cả en locale.
  • None Backend hoàn toàn không bị ảnh hưởng.
  • None Các component khác không bị ảnh hưởng.

9. Các bước thực hiện (qcode)

  1. Thêm entry cho codex_exec vào SIDEBAR_CATEGORIES trong styleUtils.ts
  2. Thêm translation key vào translation.json (en)
  3. Chạy make format_frontend
  4. Build frontend & verify với qstart
