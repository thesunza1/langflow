Root Cause Analysis: Build Hanging / Freezing

File phân tích tại: src/backend/base/langflow/api/build.py (834 dòng), src/lfx/src/lfx/graph/graph/base.py (2566 dòng), src/lfx/src/lfx/graph/vertex/base.py (952 dòng), src/frontend/src/controllers/API/api.tsx (414 dòng), src/frontend/src/stores/flowStore.ts (1571 dòng)

Tổng Quan Luồng Build

1. Frontend POST /build/{flow_id}/flow → nhận job_id
2. Backend: start_flow_build → tạo queue + event_manager + background task
3. Backend: generate_flow_events → tạo graph, sort vertices → _run_vertex_build
4. _build_vertex → graph.build_vertex() → vertex.build() → gọi LLM
⏳ KHÔNG timeout — nếu LLM không response, treo mãi
5. asyncio.gather(*tasks) chờ tất cả vertices
⏳ Nếu 1 vertex treo, tất cả đợi — không có tổng timeout
6. Frontend GET /events/{job_id} → consume_and_yield chờ queue
⏳ await queue.get() — KHÔNG có timeout
7. Frontend reader.read() chờ stream data
⏳ await reader.read() — KHÔNG có timeout

Phân Tích Root Cause

Tất cả các điểm có thể gây HANG

CRITICAL backend

1. Không có timeout trên vertex.build()

File: src/lfx/src/lfx/graph/vertex/base.py:837-843

for step in self.steps:
    if step not in self.steps_ran:
        await step(user_id=user_id, event_manager=event_manager, **kwargs)
        # ^ KHÔNG có timeout — nếu LLM API không response, build treo vĩnh viễn
        self.steps_ran.append(step)

Hậu quả: Toàn bộ chain bị block. Streaming response không gửi sentinel → frontend chờ mãi. Sentinels chỉ được gửi khi exception (qua _guarded_task), nhưng nếu await step() không bao giờ trả về thì exception cũng không có.

Fix: Thêm asyncio.wait_for(step(...), timeout=...) với timeout configurable.

CRITICAL backend

2. Không có timeout trên asyncio.gather tổng thể

File: src/backend/base/langflow/api/build.py:704

async def _run_vertex_build() -> None:
    tasks = []
    for vertex_id in ids:
        task = asyncio.create_task(build_vertices(vertex_id, graph, event_manager, vertex_timedeltas))
        tasks.append(task)
    try:
        await asyncio.gather(*tasks)  # ← KHÔNG có timeout tổng thể!
    except asyncio.CancelledError:
        ...

Hậu quả: Nếu 1 vertex treo vì Fix 1 không được áp dụng, gather chờ tất cả → build không bao giờ kết thúc → không gửi sentinel → frontend đứng.

Fix: Thêm timeout tổng thể asyncio.wait_for(gather(...), timeout=TOTAL_BUILD_TIMEOUT_S).

CRITICAL backend stream

3. Không có timeout trên queue.get() trong streaming + thiếu sentinel khi hoàn thành bình thường

File: src/backend/base/langflow/api/build.py:283

async def consume_and_yield() -> AsyncIterator[str]:
    try:
        while True:
            try:
                event_id, value, put_time = await queue.get()
                # ^ BLOCK vô thời hạn nếu build không gửi event/sentinel
                if value is None:
                    break  # sentinel — chỉ được gửi khi có exception (qua _guarded_task)
                yield value.decode("utf-8")
            except Exception as exc:
                break

Vấn đề sentinel: _guarded_task (job_queue/service.py:171) chỉ gửi sentinel (None, None, time.time()) khi có EXCEPTION. Khi build hoàn thành bình thường, event_manager.on_end() chỉ gửi event JSON {"event": "end"} (value là bytes, KHÔNG phải None). consume_and_yield() đọc event này, yield nó, rồi quay lại queue.get() — và BLOCK MÃI MÃI.

Hậu quả: HTTP connection không bao giờ đóng → frontend reader.read() cũng treo mãi.

Fix: Sau khi generate_flow_events hoàn thành, cần gửi sentinel (None, None, time.time()) vào queue — kể cả khi thành công. Thêm timeout với heartbeat cho queue.get().

HIGH frontend

5. buildController bị thay thế khi build mới (global state)

File: src/frontend/src/stores/flowStore.ts:1311

setBuildController: (controller) => {

Hậu quả: Nếu build cũ đang chạy và build mới bắt đầu, controller cũ bị thay thế. Khi build cũ hoàn tất, không thể abort được nữa. Frontend gửi request với controller đã bị vô hiệu hóa.

Fix: Lưu controller trong closure của từng build request, không ghi đè global state. Dùng per-build AbortController.

LOW backend

7. Timeout trên asyncio.Lock acquisition

File: src/lfx/src/lfx/graph/graph/base.py:1869

Phân tích lại: Lock key là f"{flow_id_str}:{run_id}" (build.py:463) — mỗi run có lock riêng. Hai build song song dù cùng flow cũng KHÔNG contention. Lock này chỉ bị contention nếu cùng một build gọi nhiều lần (không thể xảy ra trong thực tế). Không phải nguyên nhân gây hang.

Fix: Vẫn nên có timeout cho safety, nhưng ưu tiên thấp nhất.

Sentinel Gap Analysis

PHÁT HIỆN MỚI

Sentinel chỉ được gửi qua _guarded_task khi có exception. Khi build hoàn thành bình thường, không có sentinel được gửi:

// job_queue/service.py:_guarded_task
try:
    await task_coro  # generate_flow_events hoàn thành OK
except asyncio.CancelledError:
    raise
except Exception as exc:
    # Chỉ TRƯỜNG HỢP NÀY mới gửi sentinel!
    main_queue.put_nowait((None, None, time.time()))
    raise
# KHÔNG CÓ sentinel ở normal path!

Điều này có nghĩa: ngay cả khi build chạy thành công (không hang), luồng streaming vẫn treo sau event cuối cùng. Frontend chỉ thoát được khi user refresh hoặc disconnect.

Fix: Thêm main_queue.put_nowait((None, None, time.time())) SAU KHI await task_coro thành công.

STREAMING_QUEUE_TIMEOUT = 30.0  # seconds without events → send heartbeat

async def consume_and_yield():
    try:
        while True:
            try:
                event_id, value, put_time = await asyncio.wait_for(
                    queue.get(), timeout=STREAMING_QUEUE_TIMEOUT

                if value is None:

                yield value.decode("utf-8")

Fix 4: Frontend streaming timeout

// src/frontend/src/controllers/API/api.tsx
const STREAM_TIMEOUT = 120_000; // 2 minutes without data

// Thay vì:
const { done, value } = await reader.read();

// Thành:
function withTimeout(promise, ms, signal) {
    return Promise.race([
        promise,
        new Promise((_, reject) => {
            const timer = setTimeout(() => reject(new Error('Stream timeout')), ms);
            signal?.addEventListener('abort', () => {
                clearTimeout(timer);
                reject(new DOMException('Aborted', 'AbortError'));
            });
        }),
    ]);
}
const { done, value } = await withTimeout(reader.read(), STREAM_TIMEOUT, buildController.signal);

In Scope / Out of Scope

In Scope

  • Thêm timeout configurable trên tất cả blocking operations (vertex build, gather, queue get, streaming read)
  • Gửi sentinel sau normal completion (sửa sentinel gap)
  • Per-build AbortController + per-flow isBuilding flag

Out of Scope

  • Background worker architecture cho builds (quá lớn)
  • Redis/Kafka-based job queue redesign (đã có RedisJobQueueService)
  • UI redesign cho concurrent builds (chỉ sửa logic store)
