Run from Beginning — Fix bỏ qua build khi có cache

Graph execution engine dùng cached result nên không chạy lại các bước đã build

1. Phân tích hiện trạng

1.1 Triệu chứng

  1. User chạy flow A → B lần đầu: cả A và B đều build thành công
  2. User click B → Run → Run from beginning
  3. Kết quả mong đợi: A và B đều chạy lại từ đầu
  4. Kết quả thực tế: A không chạy lại (0ms), B có chạy

1.2 Nguyên nhân gốc

Trong src/lfx/src/lfx/graph/graph/base.py, method build_vertex() (dòng 1618):

if get_cache is not None:
    cached_result = await get_cache(key=vertex.id)
    if not isinstance(cached_result, CacheMiss):
        # kết quả từ cache → skip build, dùng cached data
        should_build = False
        # ... set vertex.built, vertex.results, v.v. từ cache

Cache service được inject vào từ chat_service.get_cache(). Khi chạy lần 2, kết quả của A vẫn còn trong cache → bị skip.

Không có cơ chế phân biệt "chạy lần đầu" vs "chạy từ beginning". Cả hai đều dùng chung cache.

1.3 Data flow

1
UI gọi POST /build với start_component_id=B
2

3

4

2. Yêu cầu chức năng

FR-01: "Run from beginning" phải chạy lại tất cả vertices

  • Khi user click "Run from beginning", tất cả các node từ đầu flow đến node được chọn đều phải được rebuild
  • Không được dùng cached results từ lần chạy trước

FR-02: Incremental run vẫn hoạt động

  • Khi user chỉ chạy 1 node đơn lẻ (không phải "Run from beginning"), các upstream node vẫn dùng cache như bình thường

3. Giải pháp

Cách tiếp cận: Invalidate (xoá) cache của tất cả vertices upstream của start_component_id trước khi build.

Điều này đảm bảo:

  • Khi build từ A (full run) → mọi vertex đều không có cache → build hết
  • Khi build từ B (incremental, chỉ chạy node B) → cache của B bị xoá, các node trước đó giữ nguyên cache

Điểm sửa: Trong src/backend/base/langflow/api/build.py, hàm generate_flow_events() — trước khi sort vertices và build, thêm bước clear cache cho tất cả vertices trong vertices_to_run.

6. Scope boundary

✅ Trong scope

  • Xoá cache của các vertex upstream trước khi build
  • Chỉ áp dụng cho "Run from beginning" (không ảnh hưởng incremental run)

❌ Ngoài scope

  • Sửa frontend UI

  • Các component khác

7. Các bước thực hiện (qcode)

  1. Đọc chat_service interface để biết method xoá cache (có thể là delete_cache, clear_cache, invalidate)
  2. Sửa generate_flow_events() trong build.py — thêm clear cache cho các vertex trước khi build
  3. Kiểm tra với flow A → B: chạy lần 1, chạy lại "Run from beginning" lần 2 → cả A và B đều phải rebuild
