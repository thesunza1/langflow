Codex Exec — Real-time Log Stream

Stream output từ Codex CLI về frontend real-time trong khi chạy

1. Phân tích hiện trạng

Hiện tại CodexExecComponent.execute() dùng subprocess.run() — block cho đến khi Codex CLI chạy xong, rồi mới trả kết quả.

Vấn đề: Người dùng không thấy được tiến trình đang chạy, không biết Codex đang làm gì (đang đọc file, đang suy nghĩ, đang chạy lệnh...).

Mong muốn: Khi Codex Exec đang chạy, output log từ Codex CLI hiển thị real-time trên khung "Running" của node.

2. Giải pháp

Cơ chế: Component base class có method self.log(message) — gửi log events qua EventManager → frontend real-time.

Các bước:

  1. Đổi từ subprocess.run() (block) sang subprocess.Popen() (async streaming)
  2. Đọc stdout line-by-line, gọi self.log(line) cho mỗi dòng
  3. Vẫn giữ --output-last-message để lấy final answer cho Result output
  4. Stderr cũng stream log riêng

5. Implementation detail

import subprocess
import asyncio
import os

# 1. Start process
process = subprocess.Popen(
    cmd,
    stdout=subprocess.PIPE,
    stderr=subprocess.PIPE,
    text=True,
    bufsize=1,  # line buffered
)

# 2. Stream stdout + stderr line by line
import select
import time

start_time = time.time()
timeout_secs = int(self.timeout)

while True:
    # Check timeout
    if time.time() - start_time > timeout_secs:
        process.kill()
        raise subprocess.TimeoutExpired(cmd, timeout_secs)

    # Read a line from stdout
    stdout_line = process.stdout.readline()
    if stdout_line:
        self.log(stdout_line.rstrip(), name="codex")

    # Read a line from stderr
    stderr_line = process.stderr.readline()
    if stderr_line:
        self.log(stderr_line.rstrip(), name="codex:err")

    # Check if process finished and no more output
    if not stdout_line and not stderr_line and process.poll() is not None:
        break

process.wait()
